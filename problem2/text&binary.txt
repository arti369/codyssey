1. 텍스트 파일 (Text File)
사람이 읽고 쓸 수 있는 문자(글자)들로만 이루어진 파일입니다. 우리가 앞서 다룬 .csv 파일이나 파이썬 코드 파일(.py), 메모장 파일(.txt) 등이 여기에 속합니다. 컴퓨터는 저장된 0과 1의 데이터를 UTF-8이나 ASCII 같은 문자 규칙에 따라 글자로 번역해서 화면에 보여줍니다.

장점:

사람 친화적: 메모장 같은 기본 편집기만 열어도 내용을 바로 확인하고 수정할 수 있습니다.

높은 호환성: 윈도우, 맥, 스마트폰 등 어떤 운영체제나 기기에서도 특별한 변환 없이 쉽게 읽고 쓸 수 있습니다.

쉬운 디버깅: 데이터에 오류가 생겼을 때 눈으로 직접 보고 어디가 문제인지 파악하기 좋습니다.

단점:

비효율적인 저장 공간: 예를 들어 숫자 1234567890을 저장할 때, 컴퓨터 메모리의 이진 구조에서는 4바이트면 충분하지만 텍스트 파일은 글자 하나당 1바이트씩 총 10바이트를 차지합니다.

처리 속도 저하: 파이썬 같은 프로그램이 파일에서 숫자를 읽어와 계산을 하려면, 읽어온 '문자열'을 '숫자형(int, float)'으로 변환하는 추가 과정이 필요해 속도가 느려집니다.

2. 이진 파일 (Binary File)
사람이 아닌 컴퓨터가 처리하기 편하도록 메모리에 올라간 데이터 형태 그대로(0과 1의 원시 바이트 덩어리) 저장한 파일입니다. 이미지(.jpg), 음악(.mp3), 동영상(.mp4), 실행 프로그램(.exe) 등이 대표적입니다.

장점:

뛰어난 속도: 메모리에 있는 데이터를 그대로 복사해서 파일에 쓰고, 그대로 다시 메모리로 가져오기 때문에 변환 과정이 없어 읽고 쓰는 속도가 매우 빠릅니다.

용량 절약: 데이터를 글자로 풀어서 쓰지 않고 컴퓨터의 데이터 표현 방식 그대로 압축적으로 저장하므로 대용량 데이터를 다룰 때 파일 크기가 훨씬 줄어듭니다.

단점:

사람이 읽기 어려움: 글자 규칙이 적용되어 있지 않기 때문에 메모장으로 열면 이상한 특수문자나 외계어처럼 깨져서 보입니다. 눈으로 보고 직접 수정하는 것은 불가능에 가깝습니다.

해독 규칙 필요: 해당 파일이 어떤 순서와 규격으로 저장되었는지 정확한 구조를 알아야만(혹은 전용 프로그램이 있어야만) 데이터를 올바르게 읽어낼 수 있습니다.
